# TypeScript

基本类型

  • 类型声明

    • 类型声明是TS非常重要的一个特点

    • 通过类型声明可以指定TS变量的类型

    • 指定类型后，当为变量赋值时，TS编译器会自动检查值是否符合声明类型，符合则赋值，否则报错。

    • 简而言之，类型声明是给变量设定了类型，使得变量只能存储某种类型的值。

    • 语法：

      let 变量:类型;
let 变量：类型 = 值;
function fn(参数:类型,参数:类型):类型{
.......
}

  • 自动类型判断

    • TS拥有自动的类型判断机制

    • 当对变量的声明和赋值是同时进行的，TS编译器会自动判断变量的类型

    • 所有如果你的变量声明和赋值同时进行，可以省略类型声明。

  • 类型

    类型 例子 描述
    number 1，-33 任意数字
    string 'xxx' 任意字符串
    boolean true 布尔值
    字面量 其本身 限制变量的值就是该字面量的值
    any * 任意类型
    unknown * 类型安全的any
    vooid 空值undefined 没有值
    never 没有值 不能时任何值
    object 任意的js对象
    array [1,2] 任意js数组
    tuple [4,5] 元素，ts新增类型，固定长度数组
    enum enum{A,B} 枚举，TS新增类型

编译选项

配置tsconfig.json文件

可选配置：

  • include

    指定要编译的ts文件所在的位置

    例如include:["./src/hello/**/*"]

  • exclude

    指定排除不需要编译的ts文件所在位置

    例如：exclude:["./src/hello/**/*"]

  • extends

    定义被继承的配置文件

  • files

    指定被编译的文件列表，只有需要编译的文件少时才会用到

    例如：files:["core.ts","sys.ts"]

  • compilerOptions

    编译器的选项时配置文件中非常重要也比较复杂的配置选项。

    包含多个子选项，用来完成对编译的配置

    • 项目选项

      • target

        • 设置ts代码编译的目标版本

        • 可选值ES3(默认)、ES5、ES6/ES205、ES7/2016、ES2017、ES2018、ES2019、ES2020、ESnext

        • 实例：

          "compilerOptions":{
  "target":"ES6"
}
所有ts代码将被编译成es6版本的js代码

      • lib

        • 指定代码运行时所包含的库（宿主环境）

        • 可选值：ES5、ES6/ES205、ES7/2016、ES2017、ES2018、ES2019、ES2020、ESnext、DOM、WebWorker、ScriptHost

        • 实例：

          "compilerOptions":{
  "target":"ES6",
  "lib":["ES6","DOM"]
}

      • module

        • 设置编译后代码使用的模块化系统

        • 可选值：CommonJs、UMD、AMD、System、ES2020、ESNext、None

        • 实例：

          "compilerOptions":{
  "target":"ES6",
  "lib":["ES6","DOM"],
  "module":"CommonJS",
}

      • outDir

        • 用来指定编译后文件所在目录

          "compilerOptions":{
  "target":"ES6",
  "lib":["ES6","DOM"],
  "module":"CommonJS",
  "outDir":"./dist",
}

      • outFile

        • 指把编译后的全局作用域中的代码合并为一个文件

          "compilerOptions":{
  "target":"ES6",
  "lib":["ES6","DOM"],
  "module":"CommonJS",
  "outDir":"./dist",
  "outFile":"./dist/app.js"
}

      • allowJs

        • 是否对js文件进行编译，默认是false，不编译。

      • checkJs

        • 是否检查js代码是否符合语法规范，默认值false

      • removeComments

        • 是否移除注释，默认值是false

      • noEmit

        • 不生成编译后的文件，默认值是false

      • noEmitOnError

        • 当有错误时不生成编译后的文件，默认值false

      • alwaysStrict

        • 用来设置编译后的文件是否使用严格模式，默认是false，可以提升js性能

      • noImplicitAny

        • 是否检测隐式的any类型属性。默认是false

      • noImplicitThis

        • 不允许不明确类型的this，默认值false

      • strictNullChecks

        • 严格的检查空值，默认值是false

      • strict

        • 所有严格检查的总开关，默认值是false，不打开，true打开时，其他的严格检查就不用写了。

webpack打包ts代码

  1. 先初始化npm项目使用npm init -y

  2. 安装webpack依赖、ts依赖

    npm i -D webpack webpack-cli typescript ts-loader

  3. 添加webpack.config.js文件，并配置

    const path = require('path');
//引入html插件
const HTMLWebpackPlugin = require('html-webpack-plugin');
const { CleanWebpackPlugin } = require('clean-webpack-plugin')
//webpack中的所有配置信息都应该写在module.export中
module.exports = {
    //指定入口文件
    entry: "./src/index.ts",
    //指定打包文件所在目录
    output: {
        //指定打包文件的目录
        path: path.resolve(__dirname, 'dist'),
        //打包后文件的
        filename: "bundle.js"
    },
    // 指定webpack打包时使用模块
    module: {
        //指定要加载的规则
        rules: [
            {
                //test指定的是规则生效的文件
                test: /\.ts$/,
                //要使用的loader
                use: 'ts-loader',
                //排除不需要编译的文件
                exclude: /node_modules/
            }
        ]
    },
};

  4. package.json文件配置打包命令build:webpack,就可以使用命令打包

  5. 自定义html模版

    1. 添加html-webpack-plugin插件

      1. npm i -D html-webpack-plugin

    2. 在webpack.config.js配置文件中配置模版

       //配置webpack插件
    plugins: [
        new HTMLWebpackPlugin({
            template: './public/index.html'
        }),
        new CleanWebpackPlugin(),
    ],
    指定自定义htnl模版
    CleanWebpackPlugin 可以在打包时先清除文件，再打包。
    先npm i -D clean-webpack-plugin

  6. webpack热部署

    1. 安装依赖npm i -D webpack-dev-server

    2. package.json文件配置运行命令 "start": "webpack serve",就可以使用命令运行。

    3. 改变文件会自动刷新页面

  7. 使用模块化ts、js

    在在webpack.config.js配置文件中配置使用模块化

        //用来设置引用模块
    resolve: {
        extensions: ['.ts', '.js']
    }

  8. 为了兼容更低的浏览器，可以使用babel来设置兼容性处理

    1. 安装babel依赖npm i -D @babel/core @babel/preset-env babel-loader **core-js **

    2. 配置babel

      在webpackconfigjs文件中配置

       module: {
        //指定要加载的规则
        rules: [
            {
                //test指定的是规则生效的文件
                test: /\.ts$/,
                //要使用的loader
                use: [
                    {
                        //指定加载器
                        loader: "babel-loader",
                        //设置babel
                        options: {
                            //设置预定义环境
                            presets: [
                                [
                                    //指定环境插件
                                    "@babel/preset-env",
                                    //配置信息
                                    {
                                        //要兼容的目标浏览器
                                        targets: {
                                            "chrome": "88",
                                            "ie": "11"
                                        },
                                        //指定corejs版本
                                        "corejs": "3",
                                        //使用codejs的方式usage表示按需加载
                                        "useBuiltIns": "usage",
                                    }
                                ]

                            ]
                        }
                    },
                    'ts-loader'//放在后面的先执行，tsloader先把ts代码转化为特定版本的js代码
                ],
                //排除不需要编译的文件
                exclude: /node_modules/
            }
        ]
    },

    3. 指定不需要使用箭头函数，为了兼容ie11

       output: {
        //指定打包文件的目录
        path: path.resolve(__dirname, 'dist'),
        //打包后文件的
        filename: "bundle.js",
        //告诉webpack打包不要使用箭头函数，为了兼容ie11
        environment: {
            arrowFunction: false
        }
    },

接口

接口用来定义一个类结构,包含属性和方法。 

interface myInterface{
  name: string,
  age: number
}

范型

在定义函数或者类时，遇到类型不明确的就可以使用范型。

function fn<T>(a:T):T{
  return a;
}

interface Inter{
  length: number;
}
// 这个类型的范型必须实现Inter接口，才可以作为参数。
function fn3<T extends Inter>(a:T):number{
  a.length;
}
