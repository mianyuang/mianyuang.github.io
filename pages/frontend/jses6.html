# JavaScript进阶：ES6-ES11

# let声明变量

  1. 声明的变量不能重复，var时可以声明重复的变量。

    let a = 1

  2. 块级作用域，只在块里面有效{}。

  3. 不存在变量提升，var是会变量提升。

    console.log(song)//不会报错
var song='xxx'

console.log(song) //报错
let song='xxxx'

  4. 不影响作用域链。即在let定义的作用域内使用函数作用域也可以使用let定义的变量

    {
  let school='尚硅谷';
  function fn(){
    console.log(school)
  }
  fn()
}

    在循环时尽量使用let.

const声明常量

  1. 一定要赋初始值

  2. 一般常亮建议使用大写

  3. 常亮的值不允许修改

  4. 也是块级作用域

  5. 对于数组和对象的修改，不算对常量的修改，不会报错。

工作原理

  1. 创建一个对象指针，指向当前数据结构的起始位置

  2. 第一次调用对象的next方法，指针自动指向数据结构的第一个成员

  3. 接下来不断调用next方法，指针一直往后移动，直到指向最后一个成员

  4. 每调用next方法返回一个包含value和done的属性对象

//自定义遍历实现iterator属性
let person={
  name:'xxx',
  love:['a','b','c','d']
  //实现iterator属性
  [Symbol.iterator](){
    let index=0;
    let _this=this;
    return{
      next:function(){
        if(index < _this.love.length){
          const result = {value:_this.love[index],done:false}
          index++;
          return result
        }else{
          return { value:undefined,done:true}
        }
      }
    }
  }
}

Promise异步编程

promise有三种状态，第一种成功，第二种失败，第三种初始化。

当调用resolve函数时，是成功状态，reject函数时是失败状态。

then方法两个参数接收两个方法，第一个是成功状态的方法，第二个是失败的方法，then方法可以返回promise的对象。

const p = new Promise(function(resolve,reject){
    let data=‘xxxx’
})
p.then(function(value){成功},function(value){失败})

  2. 指数操作符

ES8的新特性

  1. async和await

    两者结合可以让异步代码像同步代码一样。

    1. async函数

      1. async函数的返回值为promise对象

        1. 返回的结果不是一个promise类型对象，就会返回promise的成功状态。

        2. 抛出异常返回的是失败的promise失败状态。

        3. 返回的是一个promise对象

      2. promise对象的结果由async函数执行的返回值决定

    2. await表达式

      1. await必须用在async函数中

      2. await右侧的表达式一般为promise对象

      3. await右侧不是promise对象，则返回该值

      4. await右侧是promise成功状态对象，则返回成功状态的promiseResult的值

      5. await的promise失败了，就会抛出异常，需要通过try.catch捕获处理。

  2. Object.values和Object.entries

    1. Object.values方法返回一个给定对象的所有可枚举属性值的数组。

    2. Object.entries方法返回一个给定对象自身可遍历属性[key，value]的数组。

ES11新特性

  1. 私有属性 前面加#声明的属性是私有属性

  2. promise的allSettled方法

    1. 返回的是成功的promise对象，不管里面的peomise有成功还是失败的。

    2. 接收的参数是一个promise对象的数组。

    3. all方法和他类型，但是这个all方法只要有一个错误，返回就是失败的promise。

  3. 可选链操作符

    ?.为可选链操作符号，对象属性层层嵌套时，为了防止传入的对象为空导致出错，可选择此符号操作。

  4. 动态引入import

    在需要引入外部依赖时，动态引入。

    可以使用import(需要引入的模块),返回的是一个promise对象，就可以then回调调用。
