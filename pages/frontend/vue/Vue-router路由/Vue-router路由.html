# Vue-router路由

# 路由

  1. 理解：一个路由（route）就是一组映射关系，多个路由需要路由器（router）进行管理。

  2. 前端路由：key是路径，value是组件

# 基本使用

  1. 安装vue-router，命令npm i vue-router

  2. 应用插件：Vue.use(VueRouter)

  3. 编写router配置项

    //引入VueRouter
import VueRouter from 'vue-router'
//引入组件
//创建router实例对象，并去管理一组一组路由规则
const router = new VueRouter({
  routes:[
    {
      path:'/about',
      component:.........
    }
  ]
})

  4. 实现切换（active-class可配置高亮样式）

    <router-link active-class='active' to="/about">About</router-link>

  5. 指定展示位置

    <router-view></router-view>

# 几个注意点

  1. 路由组件通常存放在pages文件夹，一般组件通常放在components文件夹。

  2. 通过切换，隐藏了路由组件，默认是被销毁的，需要的时候在去挂载。

  3. 每个组件都有自己的$route属性，里面存储着自己的路由信息。

  4. 整个应用只有一个router，可以通过组建的$router属性获取到。

# 多级路由

  1. 配置多级路由项，使用children配置项：

    routes:[
  {
    path:'about',
    component:About,
    children:[
      {
        path:'news',
        component:News
      }
    ]
  }
]

  2. 跳转（需要写完整路径）

    <router-link active-class='active' to="/about/news">News</router-link>

# 路由的query参数

  1. 传递参数

    // 第一种 to字符串的写法
<router-link active-class='active' :to="/about/news?id=666&title=nihao">News</router-link>

//第二种写法 to对象写法
<router-link active-class='active' :to="{
  path:'/about/news',
  query:{
    id:666,
    title:'你好'
  }
}">News</router-link>

  2. 接收参数

    $route.query.id
$route.query.title

# 命名路由

  1. 给路由命名

    routes:[
  {
    path:'about',
    component:About,
    children:[
      {
        path:'news',
        name:'hello',
        component:News
      }
    ]
  }
]

  2. 简化跳转

    <router-link active-class='active' :to="{name:'hello'}">News</router-link>
//命名传递参数
<router-link active-class='active' :to="{
  name:'hello',
  query:{
    id:666,
    title:'你好'
  }
}">News</router-link>

# 路由的params参数

  1. 配置路由，声明接收params参数

    routes:[
  {
    path:'about',
    component:About,
    children:[
      {
        path:'news/:id/:title', //使用占位符声明接收params参数
        name:'hello',
        component:News
      }
    ]
  }
]

  2. 传递参数

    // to字符串写法
<router-link active-class='active' :to="/news/666/nihao">News</router-link>
//to对象传递参数
<router-link active-class='active' :to="{
  name:'hello',
  params:{
    id:666,
    title:'你好'
  }
}">News</router-link>

📌特别注意：路由携带params参数时，若使用to对象写法，则不能使用path配置项，必须要使用name配置。

# 路由的props配置

让路由组件更方便的接收参数

routes:[
  {
    path:'about',
    component:About,
    children:[
      {
        path:'news/:id/:title', //使用占位符声明接收params参数
        name:'hello',
        component:News,
        //第一种写法，props值为对象，该对象中所有的key-value的组合最终都会通过props组件传递给当前组件
        props:{a:900}
        //第二种写法，props值为布尔值，布尔值为true，则把路由接收到的所有params参数通过props传递给当前组件
        props:true
        //第三种写法，props值为函数，该函数返回的对象中每一组key-value都会通过props传给当前组件
        props(route){
          return {
            id:route.query.id,
            title:route.query.title
          }
        }
      }
    ]
  }
]

# 编程式路由导航

  1. 作用：不借助router-link实现路由跳转，让路由跳转更加灵活

  2. 具体编码实现

    this.$router.push({
  name:'xiangqing',
  params:{
    id:xxxx,
    title:xxxxx
  }
})
this.$router.replace({   //替换方式
   name:'xiangqing',
  params:{
    id:xxxx,
    title:xxxxx
  }
})

# 缓存路由组件

  1. 作用：让不展示的路由组件保持挂载，不被销毁。

  2. 具体编码实现：

    <keep-alive include="news">
  <router-view></router-view>
</keep-alive>

# 两个新的生命周期钩子函数

  1. 作用：路由组件所独有的两个钩子，用于捕获路由组件的激活状态。

  2. 具体名字：

    1. activated 路由组件被激活触发。

    2. deactivated路由组件失活时触发。

# 路由守卫

  1. 作用：对路由进行权限控制

  2. 分类：全局守卫、独享守卫、组件内守卫

  3. 全局守卫

    //全局前置守卫，初始化时执行，每次路由切换时执行
router.beforeEach(to,from,next)=>{
  ..............
}
//全局后置守卫，初始化时执行，每次路由切换后执行
router.afterEach(to,from)=>{
...............
  if(to.meta.title){
    document.title=to.meta.title
  }else{
    document.title='vue_test'
  }
}

  4. 独享守卫

    // 某一个路由独享的守卫 写在想要使用的路由里面
router.beforeEnter(to,from,next)=>{
  ..............
}

  5. 组件内守卫

    //路由进入之前触发的函数
beforeRouteEnter(to,from.next){
.....
}
//路由离开时触发的函数
beforeRouteLeave(to,from.next){
.....
}
