# VUE3快速上手

# 一、常用Composition API（常用组合API）

  1. setup

    1. 理解：vue3中一个新的配置项，和data，methods配置项一样，setup是一个函数。

    2. setup是所有Composition API 表演的舞台。

    3. 组件中所用到的数据，方法等等，都要在setup中配置。

    4. setup函数两种返回值：

      1. 若返回一个对象，则对象中属性，方法都可以在vue模版中直接使用。

      2. 若返回的是一个渲染函数，则可以自定义渲染内容。

  2. ref函数

    • 作用：定义一个响应式的数据。

    • 语法：const xxx=ref（initValue）

      • 创建一个包含响应式数据的引用对象(reference对象)

      • JS中操作数据：xxx.value

      • 模版中读取数据，不需要value，直接<div></div>

    • 备注：

      • 接收的数据可以是基本类型，也可以是对象类型。

      • 基本类型的数据：响应式任然是靠Object.defineProperty()的get和set函数。

      • 对象类型的数据： 内部求助了vue3新的一个函数—reactive函数

  3. reactive函数

    • 作用：定义一个对象类型的响应式数据（基本类型不需要用它，需要ref函数）

    • 语法：const 代理对象=reactive(原对象)接收一个对象或者数组，返回一个代理对象。

    • reactive定义的响应式数据是深层次的。

    • 内部基于ES6的Proxy实现，通过代理对象操作原对象内部数据进行操作。

  4. Vue3中的响应式原理

    1. Vue2中的响应式

      • 实现原理

        • 对象类型：通过Object.defineProperty()对属性的读取，修改进行拦截（数据劫持）

        • 数组类型：通过重写更新数组中一系列方法进行拦截。

          Object.defineProperty(data，‘count’，{
  get(){}
  set(){}
})

      • 存在问题

        • 新增属性，删除属性，界面不会更新

        • 直接通过下标修改数组，界面不会自动更新。

        • 可以通过以下方法实现响应式更新

          this.$set(要操作的数据,修改的值)
Vue.set()

    2. Vue3中的响应式

      • 实现原理：

        • 通过Proxy（代理）：拦截对象中任意属性的变化，包括：属性的读写、属性的添加、属性的删除等。

        • 通过Reflect（反射）：对被代理对象属性进行操作。

        new Proxy(data,{
  //读取属性
  get(target,proName){
    return Reflect.get(target,proName)
  }
  // 拦截设置属性或添加新属性
  set(target,propName,value){
    return Reflect.set(target,propName,value)
  }
  // 拦截删除属性
  deleteProperty(target,propName){
    return Reflect.deleteProperty(target,PropName)
  }
})

  5. Reactive对比Ref

    • 从定义数据角度对比：

      • ref用来定义基本数据类型

      • reactive用来定义对象或者数组数据类型。

      • 备注：ref也可以定义对象或者数组数据类型，内部会借助reactive转为代理独享。

    • 从原理角度对比：

      • ref主要是通过Object.defineProperty的get和set来实现响应式。

      • reactive主要是通过windows的proxy代理对象get和set、deleteProperty方法通过反射对象Reflect来操作内部数据。

    • 从使用角度对比：

      • ref数据定义：操作数据需要.value，读取数据时模版中直接读取不需要.value。

      • reactive定义的数据：操作和读取不需要.value。

  6. setUp的两个注意点

    1. setup的执行时机

      1. 在beforeCreated之前执行一次，this是undefined。

    2. setup参数

      1. props：值为对象，包含：组件外部传递过来，且组件内部声明接收了的属性。

      2. context：上下文对象

        1. attrs：值为对象，包含：组件外部传递过来，但没有在props配置中声明的属性，相当于this.$attrs。

        2. slots：收到的插槽内容，相当于$slots.

        3. emit：分发自定义事件的函数，相当于$emit。

  7. 计算属性与监视

    1. computed函数

      1. 与Vue2中computed配置功能一致。

      2. 写法

        //组合式api需要引入compound
import {reactive,computed} from 'vue'
setup(){
  let person=reactive({
    firstName:'zhang',
    lastName:'san',
  })
  //简写计算属性
  person.fullName = computed(()=>{
    return person.firstName+'-'+person.lastName;
  })
  //完整写法
  person.fullName=computed({
    get(){
      return person.firstName+'-'+person.lastName;
    }
    set(value){
      const namneArr = value.split('-');
      person.firstName=nameArr[0];
      person.lastName=nameArr[1];
    }
  })
  return{
    person
  }
}

    2. watch函数

      1. 与vue2中watch配置功能一致。

      2. 两个小坑：

        1. 监视reactive定义的响应式数据时：oldvalue无法正确获取，强制开启了深度监视(deep配置失效)

        2. 监视reactive定义的响应式数据中某个属性(对象)时，deep配置有效。

      /组合式api需要引入compound
import {reactive,computed,ref} from 'vue'
setup(){
  let name=ref('zhangsan')
  let age = ref(18)
  let person=reactive({
    name:'zhansgan',
    age:18,
    job:{
      salary:'20k'
    }
  })
  //情况一：监视ref定义的响应式数据
  watch(name,(newValue,oldValue)=>{
    console.log('name值变化了')
  },{immediate})
  // 情况二：监视ref定义的多个响应式数据
  watch([name,age],(newValue,oldValue)=>{
    console.log('name值变化了')
  },{immediate})
  // 情况三：监视reactive定义的响应式数据---全部属性
  watch(person,(newValue,oldValue)=>{
    console.log('name值变化了')
  },{immediate:true,deep:true})//deep配置无效，oldValue无法正确获取
  // 情况四：监视ractive定义的响应式数据---某个属性
  watch(()=>person.job,(newValue,oldValue)=>{
    console.log('name值变化了')
  },{immediate:true,deep:true})
  // 情况五：监视reactive定义的响应式数据---某些属性
   watch([()=>person.job,()=>person.age],(newValue,oldValue)=>{
    console.log('name值变化了')
  },{immediate:true,deep:true})
  return {
    name
  }
}

    3. watchEffect函数

      • watch的套路是：既要指明监视的属性，也要指明监视的回调。

      • watchEffect套路是：不用指明监视哪个属性，监视的回调中用到哪个属性，那就监视哪个属性。

      • watchEffect有点像computed：

        • 但computed注重的是计算出来的值，所以必须写返回值。

        • 而watchEffect更注重的是过程，所以不用写返回值。

      //watchEffect所指定的回调中用到的数据只要发生变化，则直接重新执行回调
watchEffect(()=>{
  const x1=sum.value
  const x2=person.age
  console.log('回调函数执行了')
})

  8. VUE3生命周期

    • 和vue2生命周期差不多，都有beforeCreate、created、beforeMount、mounted。

    • 不同点是beforeDestroy和destroyed变成了beforeunmounted和unmounted。

    • 生命周期钩子函数可写成配置项模式，也可以在setup()函数里面声明。

      beforeCreated()=====>setup()
created()=====>setup()
beforeMount()=====>onBeforeMounted()
mounted()=====>onMounted()
.........

  9. 自定义Hook函数

    1. 什么是Hook？本质上是一个函数，把setup函数中使用的组合式API进行了封装

    2. 类似于vue2中的mixin混入。复用数据和方法。

    //定义一个hooks文件夹，定义一个文件叫userPoint.js 
import {reactive,onBeforeMounted} from 'vue'
export default (){ //实现获取坐标点
  let point = reactive({x:0,y:0})
  onBeforeMounted(()=>{
    windows.addListener((event)=>{
      point.x = event.pageX
      point.y = event.pageY
    })
  })
  return point
}
//在 setup函数里使用
import usePoint from 'userPoint.js'
setup(){
  let point = usePoint()
  return{
    point
  }
}

  10. toRef

    • 作用：创建一个ref对象，其value值指向另一个对象中的某个属性。

    • 语法：const name=toRef(person,'name')

    • 应用：要将响应式对象中的某个属性单独提供给外部使用时。

    • 扩展：torefs与toRef功能一致，但可以批量创建多个ref对象，语法toRefs(person).

# 二、其他组合式API

  1. shallowReactive与shallowRef

    • shallowReactive只处理对象最外层属性的响应式。

    • shallowRef：只处理基本数据类型的响应式，不进行对象的响应式处理。

  2. readonly与shallowReadonly

    • readonly：让一个响应式数据变成只读的。（深只读）

    • shallowReadonly：让一个响应式数据变成只读。(浅只读)

  3. toRow与markRaw

    • toRow：

      • 作用：将一个由reactive生成的响应式对象转化为普通对象。

      • 使用场景：用于读取响应式对象对应的普通对象，对这个对象的所有操作，不会引起页面更新。

    • markRaw：

      • 作用：标记一个对象，使其永远不会成为响应式对象。

      • 应用场景：

        • 有些值不应设置为响应式：例如复杂的第三方类库。

        • 当渲染具有不可变数据源的大列表时，跳过响应式转化可以提高性能。

  4. customRef

    • 作用：创建一个自定义的ref，并对其依赖项跟踪和更新触发进行显示控制。

    • 实现防抖效果

  5. provide与inject

    • 作用：实现祖孙组件间通信

    • 套路：父组件中有一个provide选项来提供数据，子组件有一个inject选项卡来接收数据。

    • 具体写法：

      1. 祖组件中：

        
setup(){
  .......
  let car = reactive({name:'本次活动',price:'40万'})
  provide('car',car)
}

      2. 孙组件中：

        
setup(){
.......
const car = inject('car')
return {car}
}
