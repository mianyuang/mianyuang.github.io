# 集成eslint

安装eslint依赖

npm i eslint -D

初始化eslint

npx eslint --init

按照下面的选项进行选择

package.json文件添加一个lint命令

// lint当前项目中的文件并且开启自动修复
"lint": "eslint . --ext .vue,.js,.ts,.jsx,.tsx --fix",

运行lint命令，出现Paarsing error，因为没有默认解析vue文件，需要配置解析器解析

在eslintrc配置文件中增加parser来解析vue文件

// 解析vue文件
"parser":"vue-eslint-parser",

运行lint命令，出现新问题，原因是vue3特性不被eslint支持，改为使用vue3项目的规划集，使用vue3-recommend

{
    "extends": [
        "eslint:recommended",
        "plugin:vue/essential",//删除
        "plugin:vue/vue3-recommended",//增加
        "plugin:@typescript-eslint/recommended"
    ],
    "env": {
        "browser": true,
        "es2021": true,
        "node": true,
        // 开启setup语法糖环境 解决defineProps为定义问题
        "vue/setup-compiler-macros":true
    }
}

# 添加VS Code eslint插件

在vscode插件中安装eslint插件

在项目根目录下创建一个.vscode/settings.json文件，开启保存代码时自动修复。

{
  "editor.codeActionsOnSave": {
    "source.fixAll": true
  }
}

# 集成Prettier

安装 pretter依赖

npm i prettier -D

添加配置文件.prettierrc.js，添加配置

// .prettierrc.js
module.exports = {
  printWidth: 80, //单行长度
  tabWidth: 2, //缩进长度
  useTabs: false, //使用空格代替tab缩进
  semi: false, //句末使用分号
  singleQuote: true, //使用单引号
}

在package.json文件中添加命令，进行格式化代码

{
  "format": "prettier --write ./**/*.{vue,ts,tsx,js,jsx,css,less,scss,json,md}"
}

在vscode中添加prettier插件完成保存自动格式化

{
   "editor.formatOnSave": true, // 开启自动保存
   "editor.defaultFormatter": "esbenp.prettier-vscode", // 默认格式化工具选择prettier
}

# 解决eslint和Prettier冲突

发现eslint提示错误，修改后，又恢复之前的报错，自动修复失效了。因为eslint负责美化代码和prettier冲突了，需要 用 eslint-config-prettier 提供的规则集来覆盖掉eslint冲突的规则，并用eslint-plugin-prettier来使eslint使用prettier的规则来美化代码。

安装依赖

npm i eslint-config-prettier eslint-plugin-prettier -D

在 .eslintrc.json中extends的最后添加一个配置:

"plugin:prettier/recommended" // 新增，放最后

# 配置husky+lint-staged进行提交代码检查

lint-staged 是一个只检测git暂存区的lint工具，husky是一个用来给我们的项目添加git hook的工具，git hook是进行git操作会触发的脚本，例如：提交的时候会触发pre-commit钩子,输入提交信息会触发commit-msg钩子。 我们用husky安装pre-commit钩子，我们就可以在进行git commit操作的时候，运行我们的脚本来检测待提交的代码是否规范，便可以只对暂存区的文件进行检查。

安装依赖

npm i husky lint-staged -D

在package.json文件中添加一条脚本命令

 "prepare": "husky install"

prepare脚本会在 yarn install 之后自动运行，这样clone了你的项目之后会自动安装husky,这里由于我们已经运行过 yarn install 了，所以我们需要手动运行一次yarn run prepare,然后我们就会得到一个目录.husky。

为git仓库添加一个pre-commit钩子，运行命令

npx husky add .husky/pre-commit "npx --no-install lint-staged"

运行后，在.husky目录下生成一个pre-commit的脚本。

#!/bin/sh 
. "$(dirname "$0")/_/husky.sh"

npx --no-install lint-staged

完成后在配置lint-staged，在package.json中添加下面的配置信息

{
  "lint-staged": {
    "*.{js,vue,ts,jsx,tsx}": [
      "prettier --write",
      "eslint --fix"
    ],
    "*.{html,css,less,scss,md}": [
      "prettier --write"
    ]
  }
}

配置完成后提交到暂存区的代码会被eslint+prettier格式化和检查。
